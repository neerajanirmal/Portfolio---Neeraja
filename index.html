What I actually do

I build enterprise web applications where the UI isn't hard-coded — it's rendered from rules. Most of my work lives in React.js and Progressive Web Apps, reading custom JSON schemas to generate forms, chat modules, and document views on the fly. Underneath, I write the T-SQL that the front end actually depends on: stored procedures, views, and triggers carrying real business logic. I care about clean alignment, responsive layouts, and code the next developer can actually reuse.
